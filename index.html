add-existing-player
Este comando es usado para agregar a un jugador que ya existe en los registros del bot (véase check-player) a un equipo.

Este comando solo puede ser usado por un líder de equipo.
change-team-clantag
Este comando es usado para cambiar el clantag de un equipo. No puede ser cambiado a NULL.

Este comando solo puede ser usado por un líder de equipo.
change-teammate-clantag
Este comando es usado para cambiar el clantag de un jugador. Puede ser usado para quitar del roster (puede ser NULL) o cambiarlo de equipo.

NULL es solo una opción, Discord no obliga a ponerlo como valor. Puede ingresar el clantag de su equipo.

Este comando solo puede ser usado por un líder de equipo.
delete-team-registry
Este comando es usado para eliminar el registro de un equipo, junto con sus jugadores, de la base de datos del bot. Elimina los datos que usa el bot para el conteo de la tasa de victorias.

Este comando solo puede ser usado por el líder del equipo que registró el equipo.
delete-teammate-registry
Este comando es usado para eliminar el registro de un jugador, junto a los datos que usa el bot para realizar el conteo de la tasa de victorias, de la base de datos del bot. Puede ser NULL.

NULL es solo una opción, Discord no obliga a ponerlo como valor. Puede ingresar el clantag de su equipo.

Este comando solo puede ser usado por el líder del equipo que registró el equipo.
personal-stats-tier10
Este comando es usado para consultar la tasa de victorias en tier 10 de un jugador.

Este comando NO registra a un jugador. Este revisa si el jugador tiene registros en el bot y hace un cálculo de las estadísticas en tier 10. Para poder hacer conteo, vea single-teammate-registration.

Este comando puede ser usado por cualquier persona.
roster
Este comando es usado para mostrar la estructura completa de un equipo junto a la tasa de victorias de cada jugador y de todo el equipo. La tasa de victorias del equipo es un aproximado y puede llegar a variar. No puede ser NULL.

Este comando NO registra el equipo ni consulta a la API de Wargaming para conocer la tasa de victorias de un equipo. Para realizar un conteo, vea team-registration.

Este comando puede ser usado por cualquier persona.
single-teammate-registration
Este comando es usado para registrar a un jugador en la base de datos del bot. Puede ser NULL si solo quiere saber la tasa de victorias en tier 10 de un jugador sin equipo. También puede usarse para agregar jugadores que no estén en los registros del bot al roster de un equipo.

NULL es solo una opción, Discord no obliga a ponerlo como valor. Puede ingresar el clantag de su equipo.

Este comando puede ser usado por cualquier persona, en caso de solo querer conocer la tasa de victorias en tier 10, o por un líder de equipo para agregar un nuevo jugador que no esté en los registros del bot (véase check-player para verificar si existe o no el jugador a registrar).
team-registration
Este comando es usado para registrar a un equipo en la base de datos del bot. Son 10 en total; 7 obligatorios, 3 opcionales que pueden ser agregados con el tiempo (véase single-teammate-registration). Se necesitará un clantag (no puede ser NULL) y región del equipo.

Quien realice el registro del equipo, será el líder y podrá realizar cambios en la estructura del equipo. Esto no significa que podrá hacer cambios en los registros de otros equipos; solo podrá hacerlos en el equipo que usted registre. En cada comando podrás saber si puede ser usado por cualquiera o solo un líder de equipo. Puede ser usado por cualquier persona.
team-stats
Este comando es usado para consultar la tasa de victorias de un equipo. La tasa de victorias es un aproximado y puede llegar a variar. El clantag no puede ser NULL.

Este comando NO registra el equipo ni consulta a la API de Wargaming para conocer la tasa de victorias de un equipo. Para realizar un conteo, vea team-registration.

Puede ser usado por cualquier persona.
